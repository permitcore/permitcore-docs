## 1. Sign up for an API key

Magic-link signup at <a href="https://permitcore.io/signup">permitcore.io/signup</a>.
No credit card required for the free tier.

After signing in, your API key lives at <a href="https://permitcore.io/account/api-keys">Account → API Keys</a>.

Keys can be created and rotated via either the
<a href="https://permitcore.io/account/api-keys">account dashboard</a> or
the [programmatic key endpoints](/api-reference/keys).

## 2. Set up your environment

Export your key. Never commit it — treat it like a database password.

```bash
export PERMITCORE_API_KEY=pk_...
```

## 3. Make your first request

Your first call is `/v1/permits` — the core listing endpoint. Fetch recent New
York City permits (add `&cohort=<segment>` to filter to a segment):

<CodeGroup>

```bash curl
curl "https://api.permitcore.io/v1/permits?jurisdiction=nyc&per_page=10" \
  -H "Authorization: Bearer $PERMITCORE_API_KEY" \
  -H "Accept: application/json"
```

```js Node.js
const res = await fetch(
  "https://api.permitcore.io/v1/permits?jurisdiction=nyc&per_page=10",
  {
    headers: {
      Authorization: `Bearer ${process.env.PERMITCORE_API_KEY}`,
      Accept: "application/json",
    },
  },
);
const data = await res.json();
console.log(data.items);
```

```python Python
import os
import httpx

r = httpx.get(
    "https://api.permitcore.io/v1/permits",
    params={"jurisdiction": "nyc", "per_page": 10},
    headers={
        "Authorization": f"Bearer {os.environ['PERMITCORE_API_KEY']}",
        "Accept": "application/json",
    },
)
r.raise_for_status()
print(r.json()["items"])
```

</CodeGroup>

## 4. Read the response

You'll get back a page of permits in `items`, plus `pagination`:

```json
{
  "items": [
    {
      "canonical_uid": "1:BT-202601090",
      "jurisdiction_slug": "nyc",
      "address_full": "82 ASP 1ST FLOOR, Manhattan, NY 10009",
      "permit_type_normalized": "alteration",
      "work_class": "alteration_minor",
      "cohort": "commercial_alteration",
      "valuation_usd": 18400,
      "issued_date": "2026-05-27"
    }
  ],
  "pagination": { "page": 1, "per_page": 10, "has_more": true }
}
```

Each permit carries a rich, structured record — address + geocode,
classification (`cohort`, `work_class`, `permit_type_normalized`), scope,
valuation, contractor, dates, data-quality flags, and enrichment. See
[GET /v1/permits](/api-reference/permits) for the full field list and every
query parameter. Add `&cohort=<segment>` to filter to a segment, or paginate
with `&page=`.

Want the per-segment counts for a jurisdiction instead of the permit rows? Use
[GET /v1/jurisdictions/{slug}/cohorts/distribution](/api-reference/jurisdictions/cohort-distribution).
