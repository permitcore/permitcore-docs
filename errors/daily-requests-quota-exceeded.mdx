The daily request quota for your tier is exhausted.

```json 429 application/problem+json
{
  "type": "https://docs.permitcore.io/errors/daily-requests-quota-exceeded",
  "title": "Daily request quota exceeded",
  "status": 429,
  "detail": "…",
  "instance": "/v1/permits",
  "request_id": "req_1f07fb45782b445e9f6312ba23921234"
}
```

Every PermitCore error is [RFC 9457](https://www.rfc-editor.org/rfc/rfc9457)
`application/problem+json`. **Branch on `type` (a stable URI)**, not on `title`
or `detail` (human-readable, may change).

## When it occurs

Your key exhausted its daily request ceiling. The 429 body carries `quota_kind: "daily"`, `limit`, `tier_name`, and `reset_at` (next UTC midnight).

## How to fix

Sleep on `Retry-After` / `retry_after_seconds` (the full daily ceiling restores at the next UTC midnight) or upgrade your tier. See [Rate limits](/concepts/rate-limits).

**Retryable** — retry with exponential backoff (respect `Retry-After` on `429`/`503`).

See the full [error catalog](/errors) for the response shape and retry matrix.
Every response carries an `x-request-id` header — include it in any support request.
