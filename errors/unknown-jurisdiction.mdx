The jurisdiction slug was not found.

```json 404 application/problem+json
{
  "type": "https://docs.permitcore.io/errors/unknown-jurisdiction",
  "title": "Unknown jurisdiction slug",
  "status": 404,
  "detail": "…",
  "instance": "/v1/jurisdictions/xx/cohorts/distribution",
  "request_id": "req_1f07fb45782b445e9f6312ba23921234"
}
```

Every PermitCore error is [RFC 9457](https://www.rfc-editor.org/rfc/rfc9457)
`application/problem+json`. **Branch on `type` (a stable URI)**, not on `title`
or `detail` (human-readable, may change).

## When it occurs

The slug is not present in the `jurisdictions` table. On `/v1/jurisdictions/{slug}/cohorts/distribution` this is a `404`; on `/v1/territory/rollup` an unknown slug in a filter returns `422` instead.

## How to fix

Use a canonical slug (any slug that returns `200` from the cohort-distribution endpoint). See [Jurisdictions](/concepts/jurisdictions).

**Not retryable** — fix the request before resubmitting; retrying unchanged returns the same error.

See the full [error catalog](/errors) for the response shape and retry matrix.
Every response carries an `x-request-id` header — include it in any support request.
