One or more query parameters failed validation.

```json 422 application/problem+json
{
  "type": "https://docs.permitcore.io/errors/invalid-request",
  "title": "Invalid request",
  "status": 422,
  "detail": "…",
  "instance": "/v1/permits",
  "request_id": "req_1f07fb45782b445e9f6312ba23921234"
}
```

Every PermitCore error is [RFC 9457](https://www.rfc-editor.org/rfc/rfc9457)
`application/problem+json`. **Branch on `type` (a stable URI)**, not on `title`
or `detail` (human-readable, may change).

## When it occurs

A parameter had the wrong type, format, or an out-of-range value. The response body includes an `errors` array (FastAPI/Pydantic validation detail) pinpointing each offending field.

## How to fix

Read the `errors` array in the response, correct the offending parameter, and resubmit. Not retryable unchanged.

**Not retryable** — fix the request before resubmitting; retrying unchanged returns the same error.

See the full [error catalog](/errors) for the response shape and retry matrix.
Every response carries an `x-request-id` header — include it in any support request.
