---
title: "GET /v1/jurisdictions/{slug}/cohorts/distribution"
description: "Per-jurisdiction breakdown across the canonical 18 cohorts. Cached 1 hour."
api: "GET https://api.permitcore.io/v1/jurisdictions/{slug}/cohorts/distribution"
---

Returns the canonical 18-cohort permit distribution for a single jurisdiction.

The response is **always 18 keys** — cohorts with no permits are zero-filled.
Counts reflect the rolling-window total permits classified to each cohort
for that jurisdiction as of the `as_of_utc` timestamp.

## Authentication

`Authorization: Bearer <your-api-key>` — see [Authentication](/authentication).

## Path parameters

<ParamField path="slug" type="string" required>
  Lowercase jurisdiction slug. Examples: `nyc`, `la`, `houston`, `dc`. See
  [Concepts → Jurisdictions](/concepts/jurisdictions) for the coverage list.
  Returns `404` for unknown slugs and for slugs in DEPLOY-PENDING status.
</ParamField>

## Response

<ResponseField name="jurisdiction_slug" type="string">
  Echoes the path parameter for round-trip clarity.
</ResponseField>
<ResponseField name="jurisdiction_display_name" type="string">
  Human-readable metro name, e.g. `"New York City, NY"`.
</ResponseField>
<ResponseField name="as_of_utc" type="string">
  ISO 8601 UTC timestamp marking when the underlying data was last computed.
</ResponseField>
<ResponseField name="total_target_permits" type="number">
  Total permits in the rolling window across all 18 cohorts. Equal to the
  sum of `cohort_distribution` values.
</ResponseField>
<ResponseField name="cohort_distribution" type="object">
  Map of `{ cohort_slug: integer_count }` for all 18 canonical cohorts.
  Always 18 keys; zero-filled when absent.
</ResponseField>

## Errors

| HTTP | Code | Cause |
|------|------|-------|
| `401` | `missing_credentials` | No `Authorization` header |
| `401` | `invalid_token` | Key malformed / revoked |
| `403` | `insufficient_scope` | Key lacks `query` scope |
| `404` | `http_404` | Unknown slug OR DEPLOY-PENDING metro |
| `429` | `rate_limited` | Monthly quota exhausted (see [Rate limits](/concepts/rate-limits)) |
| `500` | `internal_error` | Upstream pipeline issue; retry after backoff |

## Caching

The upstream API responds with `Cache-Control: max-age=3600`. The
PermitCore site consumes this endpoint via Next.js ISR with
`revalidate: 3600`, so site-rendered pages lag the underlying pipeline
by **at most 1 hour** beyond the pipeline's own daily refresh.

For your own caching, the 1-hour value is a safe upper bound — pipeline
data only changes once per 24 hours per metro.
